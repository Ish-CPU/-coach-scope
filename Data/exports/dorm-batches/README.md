Re-run `npm run dorms:batch` after a fresh `npm run dorms:missing` to refresh.

- **Universities total in backlog:** 591

## Research task

For each row in a batch CSV, open the university's `housingUrl` (or
find one starting from `websiteUrl`) and pull the residence halls list
from the official **Housing / Residence Life** page. Produce a single
dorm-import CSV that contains every dorm discovered across the batch.

**Hard rules:**

- Use ONLY official university housing / residence life pages.
- NEVER invent dorm names. If a university's housing page doesn't list
  named residence halls, leave that university out — that's fine; the
  worksheet will surface it again on the next coverage run.
- Don't copy from third-party rankings, niche.com, dorm-reviews sites,
  or marketing pages. Official only.
- Leave optional fields blank if you can't verify them — never guess.

## Dorm import CSV format

Header (paste verbatim as the first line of your output file):

```
name,slug,universityName,city,state,roomType,bathroomType,yearBuilt,capacity,officialPageUrl,description,imageUrl,sourceUrl,sourceName,seasonYear,lastVerifiedAt
```

Required columns: `name`, `universityName`. Everything else is optional.

- `roomType` ∈ {Single, Double, Suite, Apartment}
- `bathroomType` ∈ {Private, Shared, Communal}
- `officialPageUrl` — the per-dorm housing page when one exists
- `sourceUrl` + `sourceName` — provenance (e.g. the housing index URL
  + "Stanford Residential & Dining Enterprises")
- `seasonYear` defaults to `2025-2026` if blank; `lastVerifiedAt`
  defaults to now on insert and is preserved on update.

## Import the result

Once the dorm CSV is built, upload via the admin UI:

- `/admin/import` → select type **dorms** → upload the CSV.

Or from the CLI:

```bash
npm run db:import -- dorms path/to/your-dorm-batch-NNN.csv
```

The importer is idempotent — rows are deduped per `(universityId, slug)`
and per `(universityId, normalizedName)`. Re-uploading the same CSV
never duplicates and never deletes existing dorms.

After importing a batch, run `npm run dorms:missing` to see
the coverage stats update.
