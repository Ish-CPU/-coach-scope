# RateMyU — Public Coaching Staff Collection Guide

A workflow for collecting **public, factual** coaching-staff data from **official university athletics websites only**. Output is a CSV that drops straight into RateMyU's importer (CLI or admin upload at `/admin/import`).

---

## What you may collect

For each coach on a sport's official staff page:

| Column | Required? | Source |
|---|---|---|
| `name` | yes | The exact name printed on the official staff page. |
| `slug` | optional | The URL-friendly identifier from the official profile URL (e.g. `kalen-deboer`). |
| `universityName` | yes | Must match a `University.name` already imported. |
| `sport` | yes | One of: `Football`, `Baseball`, `Softball`, `Men's Basketball`, `Women's Basketball`, `Men's Soccer`, `Women's Soccer`. (Or a base name like `Basketball` + `gender`.) |
| `gender` | optional | `Men`, `Women`, `M`, `W`, `Boys`, `Girls`. Used to disambiguate when `sport` is given as a base name. |
| `role` | yes | E.g. `Head Coach`, `Offensive Coordinator`, `Quarterbacks Coach`. See list below. |
| `level` | optional | `NCAA Division I`, `NCAA Division II`, `NCAA Division III`, `JUCO`, `NAIA`. Aliases like `D1` / `NJCAA` / `Community College` also work. |
| `conference` | optional | E.g. `SEC`, `Big Ten`, `ACC`, `Big 12`. Free text — the exact label on the official athletics site. |
| `officialProfileUrl` | optional | Direct link to this coach's profile on the staff page. |
| `sourceUrl` | yes | The official staff page itself (e.g. `https://rolltide.com/sports/football/coaches`). |
| `seasonYear` | optional | Defaults to `2025-2026` when blank. |
| `lastVerifiedAt` | optional | ISO date you verified this row. Defaults to import time when blank. |

## Allowed roles

All free text — these are the standard ones to use:

- **Head:** `Head Coach`
- **Coordinators:** `Offensive Coordinator`, `Defensive Coordinator`, `Special Teams Coordinator`
- **Position coaches (football):** `Quarterbacks Coach`, `Running Backs Coach`, `Wide Receivers Coach`, `Tight Ends Coach`, `Offensive Line Coach`, `Defensive Line Coach`, `Linebackers Coach`, `Defensive Backs Coach`
- **Other sports:** Use the title printed on the official site (`Pitching Coach`, `Hitting Coach`, `Associate Head Coach`, `Assistant Coach`, etc.)

The importer treats `role` as free text, so any official title works — but use the standard names above when possible for consistent UI grouping.

## What you may NOT collect

| Forbidden | Why |
|---|---|
| Full biographical paragraphs | Likely copyrighted by the university. Keep the `bio` column blank — the importer accepts it but you should leave it empty unless you have explicit permission. |
| Reviews, opinions, ratings, anecdotes | RateMyU reviews must be user-generated by verified accounts. The importer rejects all rating/review fields by design. |
| Personal contact info | No phone numbers, personal emails, home addresses, salaries. Office email + office phone are still PII — leave them out. |
| Player/family info | Coaches' family names, kids, etc. — none of it. |
| Content from third-party platforms | Rate My Professors, Niche, ESPN, On3, Rivals, 247Sports, social media bios — never. Only the school's own athletics site. |

## Allowed sources only

✅ **Use:**
- The university's `.edu` site.
- The official athletics site (e.g. `rolltide.com`, `texassports.com`, `gostanford.com`).
- Official roster / staff / coaches subpages (`/sports/<sport>/coaches`).

❌ **Do not use:**
- Wikipedia / fan wikis.
- News articles, press releases hosted by third parties.
- Social-media profiles (X, Instagram, TikTok bios).
- Recruiting databases (Rivals, On3, 247Sports).
- Any review or rating platform.

If a fact only exists on a non-official source, **leave the field blank**. Don't guess.

## Step-by-step workflow

1. **Confirm the university is already imported.** Open the admin import page, run the universities import first, or query the DB. The coaches importer references universities by `name`, so spelling must match exactly.

2. **Find the official athletics site.** From the university's `.edu` homepage, look for "Athletics" → the site you land on is canonical (e.g. `rolltide.com` for Alabama).

3. **Open the sport's staff page.** Usually `/sports/<sport>/coaches` or `/coaches`. That URL becomes `sourceUrl` for every row from that page.

4. **For each coach listed, copy:**
   - `name` exactly as printed
   - their official title → `role`
   - their profile link (if individual profile pages exist) → `officialProfileUrl`

5. **Fill in level + conference once per program.** They're program facts, so every coach for the same `(university, sport)` should carry the same `level` and `conference`. The importer applies them to the program (school) row.

6. **Set `seasonYear: 2025-2026` and `lastVerifiedAt: <today>`.** Stamp these so future audits know when the data was last touched.

7. **Save as CSV** with the header row exactly:
   ```
   name,slug,universityName,sport,gender,role,level,conference,officialProfileUrl,sourceUrl,seasonYear,lastVerifiedAt
   ```

8. **Import.** Either:
   ```bash
   npm run db:import -- coaches path/to/your-coaches.csv
   ```
   …or upload at `/admin/import` (admin role required).

9. **Read the result panel.** The importer reports `created / updated / skipped` plus per-row error messages (e.g. "Sport X is not supported", "University X not found"). Fix the rows it skipped and re-import — it's idempotent (`upsert` on `(school, name)`).

## Validation that runs at import time

For every row, the server enforces:

1. `name` + `universityName` + `sport` must be present.
2. `universityName` must match an existing `University` (case-sensitive). Import universities first.
3. `sport` must be one of the seven canonical sports — or a base sport that combines with `gender` into one of them. Anything else is rejected with a clear error.
4. `level` (when given) must coerce to a known `Division`. Unknown labels are rejected.
5. `officialProfileUrl` and `sourceUrl` are stored as-is — the broader importer + verification flow rejects javascript: / data: / shortener URLs elsewhere, but these directory fields don't currently call those validators (admins paste them).

Skipped rows do not abort the file — they're listed in the result panel so you can fix them and re-run.

---

## Re-imports are safe

The Coach is upserted on `(schoolId, name)`, so re-running the same CSV updates rows in place. Changing the `slug` or `role` of an existing coach just rewrites those columns. To remove a coach, delete the row from the database directly (no destructive importer action by design).
